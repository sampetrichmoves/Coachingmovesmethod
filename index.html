🔄 Flywheel — How the System Gets Smarter
Feedback loops, analytics, and error tracking. Why Arize (and tools like it) matter here.
💡 The core insight: A static protocol library is a textbook. A learning system is a coach that gets better every week. The difference is feedback loops.
🔄 The 4 Feedback Loops That Drive Improvement
Loop 1 — Client Outcome Tracking
Signal: Weekly pain scores + snack days from Progress Dashboard — compare protocol prescribed vs outcome.
Action: Protocols with less than 2pt improvement by week 4 get flagged for senior coach review.
Loop 2 — Coach Decision Logging
Signal: Regression Tree decisions + Fast Prescriber choices — which paths are used most vs least.
Action: Unused paths flag training gaps or protocol gaps. Both are fixable.
Loop 3 — Red Flag and Refer Rate
Signal: How often coaches use Refer tab + how fast they escalate.
Action: High refer rate on a specific injury = protocol needs refinement. Low rate = coaches may be missing flags.
Loop 4 — Coach Playground Insights
Signal: What senior coaches submit to the playground — edge cases, pattern breaks, surprise wins.
Action: Monthly review — senior coaches vote on insights to promote into the main protocol library. The tool literally rewrites itself.
🆕 Why Arize and Observability Tools Matter
Arize is an ML observability platform — it watches model outputs and flags when they drift, fail, or produce unexpected results.
🔍 Drift detection — if the Fast Prescriber returns the same snack for 90% of cases, something is wrong. Arize catches this.
⚠ Error tracking — bad outcomes get connected back to which prescription was given and the protocol gets flagged.
📊 Ground truth logging — linking what was prescribed to what happened. This is how the model improves vs just guessing.
